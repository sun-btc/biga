# 贡献指南

感谢你考虑为 Biga 投资研究策略 Skill 集合做贡献。本项目是 Markdown 文档型 Skill 集合，目标是沉淀可复用的投资研究框架、核验清单、风险提示和分析模板。

## 可贡献方向

欢迎提交以下类型的改进：

- 补充策略适用边界、失效条件和风险提示。
- 优化价值、成长、趋势、股息、周期、事件驱动等策略 Skill 的分析框架。
- 增加研究问题清单、复盘模板、仓位管理思路和退出纪律。
- 修正文档中的事实错误、表述歧义或过度简化内容。
- 改进 README、Skill 入口说明、目录导航和开源协作模板。

## 不接受的内容

为保持项目边界清晰，请不要提交以下内容：

- 具体荐股、喊单、买入价、卖出价、目标价或收益承诺。
- “必涨”“稳赚”“满仓”“梭哈”等确定性或诱导性表达。
- 未经核验的数据、市场传闻、内幕信息或无法追溯来源的结论。
- 鼓励高杠杆、无止损、无风险评估的交易方式。
- 自动交易脚本、代客理财逻辑或绕过合规边界的内容。

## Skill 文档建议结构

新增或大幅修改 Skill 文档时，建议保持以下结构：

```markdown
# Skill 名称

## 适用场景

## 不适用场景

## 核心分析框架

## 输入信息要求

## 分析流程

## 核验清单

## 风险信号

## 退出或停止条件

## 输出格式

## 免责声明
```

不要求所有 Skill 完全一致，但应保证读者能清楚理解：这个策略适合什么、不适合什么、如何核验、有哪些风险、什么情况下应停止使用该策略。

## Pull Request 自查清单

提交 PR 前，请确认：

- [ ] 变更内容不包含具体荐股、喊单或收益承诺。
- [ ] 变更内容不包含未经核验的数据、传闻或内幕信息。
- [ ] 涉及策略判断时，已补充适用边界和风险提示。
- [ ] 涉及高波动、周期或主题投资时，已提示回撤、拥挤度和兑现风险。
- [ ] Markdown 链接和目录路径可正常阅读。
- [ ] 修改保持文档简洁，没有引入不必要的运行环境依赖。

## Issue 提交建议

提交 issue 时，请尽量说明：

- 你希望改进的策略或文档位置。
- 当前内容存在的问题。
- 建议的修改方向。
- 是否涉及具体数据来源或外部资料。
- 是否可能引入投资误导风险。

## 金融内容边界

Biga 只用于投资研究和投资者教育，不构成任何证券、基金或其他金融产品的买入、卖出或持有建议。贡献内容应优先提供研究框架、核验清单和风险提示，而不是直接给出投资结论。
